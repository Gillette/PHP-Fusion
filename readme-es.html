INTRODUCCIÓN
PHP-Fusion es un sistema de gestión de contenidos (CMS) de código abierto, ligero y escrito en PHP 5 (compatible con PHP 4). Utiliza una base de datos MySQL para almacenar el contenido de tu sitio web e incluye un sistema de administración simple y comprensible. PHP-Fusion incluye las características más comunes que puedes encontrar en muchos otros paquetes CMS.

INSTALACIÓN
Antes de instalar PHP-Fusion, tienes que crear una base de datos MySQL. Puedes hacerlo a través del panel de control de tu servidor web o de phpMyAdmin. Asegúrate de tener a mano tus datos de acceso a MySQL, incluyendo el nombre del servidor, el nombre de usuario, la contraseña y el nombre de la base de datos, ya que tendrás que indicarlos durante la configuración.

1. Antes de transferir los archivos, debes cambiar el nombre del archivo "_config.php" (localizado en la carpeta llamada "files") y renombrarlo como "config.php".

2. Transfiere al servidor web el contenido de la carpeta "files".

3. Salvo que estés ejecutando PHP-Fusion en un servidor local, en la mayoría de los casos, tendrás que asignar los permisos CHMOD 777 a los siguientes archivos y directorios:

  • administration/db_backups/
  • images/
  • images/imagelist.js
  • images/articles/
  • images/avatars/
  • images/news/
  • images/news_cats/
  • images/photoalbum/
  • images/photoalbum/submissions/
  • forum/attachments/
  • config.php

4. Ve a tu sitio web y el archivo "setup.php" deberá iniciarse automáticamente. Si no es así, tendrás que ejecutar el archivo "setup.php" manualmente introduciendo en la barra de dirección de tu navegador la dirección URL completa de tu sitio web seguida de "/setup.php" (Ejemplo: http://www.tudominio.com/setup.php).

5. Completa el proceso de configuración siguiendo todas las indicaciones mostradas en la pantalla.

6. Inmediatamente después de completar la instalación de PHP-Fusion, debes cambiar los permisos del archivo "config.php" a CHMOD 644 y borrar del servidor web el archivo "setup.php".

ACTUALIZACIÓN DESDE V6.01.12+
Antes de actualizar, te recomendamos encarecidamente que hagas una copia de seguridad de los archivos y de la base de datos, ya que PHP-Fusion 7 es una actualización mayor comparada con sus predecesoras. Debes seguir estas instrucciones con precisión.

1. El código de la versión 6 no es directamente compatible con el de la versión 7. Antes de la llevar a cabo la actualización, deberás actualizar también las modificaciones, infusiones, paneles o temas de que dispongas u obtener actualizaciones compatibles con la versión 7. Sin embargo, es recomendable que desactives todos los códigos de terceros para evitar cualquier problema.

2. En primer lugar, debes transferir el archivo de actualización de la carpeta llamada "upgrade v601x" al directorio de administración de tu sitio web.

3. Inicia sesión en tu sitio como Super Administrador. En "Administración", dentro del apartado "Sistema", pulsa "Actualización" y, después, pulsa el botón "Actualizar". DEBES realizar la actualización a través de una serie de pasos y el proceso de actualización sólo estará completo cuando veas "Actualización de la base de datos completada".

4. MUY IMPORTANTE: Dado que esta distribución contiene una serie de cambios estructurales, algunos elementos de tu sitio no funcionarán adecuadamente hasta que no hayas actualizado todos tus archivos. DEBES transferir TODOS los archivos de la carpeta "files".

Si, a pesar de todo, tienes cualquier duda, por favor, solicita ayuda en uno de nuestros sitios de soporte, ya que en nuestra comunidad hay muchos usuarios con los conocimientos necesarios para ayudarte o aconsejarte sobre el proceso de actualización.

SUGERENCIAS DE SEGURIDAD
Aquí tienes algunos consejos útiles para ayudarte a mantener tu sitio seguro:
  • Asegúrate de que no se puede escribir en el archivo "config.php" (sus permisos deberían ser CHMOD 644).
  • Una vez que PHP-Fusion esté instalado, nunca dejes en el servidor el archivo "setup.php".
  • Asegúrate siempre de que las contraseñas de FTP y MySQL son diferentes.
  • Nunca permitas adjuntar en los foros archivos PHP, HTML, EXE o cualquier tipo de archivo de texto.
