INTRODUCTION
PHP-Fusion est un gestionnaire de contenu (C.M.S.) ultra-léger, écrit en PHP 5 (compatible PHP 4).
Il utilise une base de données MySQL pour stocker le contenu de votre site et inclut un système d'administration simple et complet. PHP-Fusion comprend les options que vous voyez dans la plupart des autres C.M.S..

INSTALLATION
Avant d'installer PHP-Fusion vous devez créer une base de données MySQL. Vous pouvez faire ceci par le panneau de contrôle de votre hébergeur ou via phpMyAdmin. Vérifiez que vous avez à portée de main les informations, en particulier le nom de l'hôte, le nom d'utilisateur, le mot de passe et le nom de la base pour les fournir durant la procédure d'installation.

1. Avant d'uploader les fichiers, renommez _config.php (dans le répertoire files) en config.php.

2. Uploadez le contenu du répertoire files avec tous ses sous-répertoires sur votre hébergeur.

3. A moins que vous soyez sur un serveur installé chez vous (et à part quelques cas particuliers comme Free en France), changez les droits (CHMOD) en 777 pour les fichiers et répertoires suivants :

  • administration/db_backups/
  • images/
  • images/imagelist.js
  • images/articles/
  • images/avatars/
  • images/news/
  • images/news_cats/
  • images/photoalbum/
  • images/photoalbum/submissions/
  • forum/attachments/
  • config.php

4. Allez sur votre site, où le fichier setup.php devrait démarrer automatiquement. Si ce n'est pas le cas, naviguez jusqu'à /setup.php. Par exemple : http://www.votresite.fr/setup.php.

5. Laissez-vous guider en répondant aux questions.

6. Immediatement après repassez le CHMOD de config.php en 644 ET effacez setup.php de votre serveur.

MISE A JOUR DEPUIS UNE VERSION V6.01.12+
Avant toute mise à jour nous vous conseillons fortement de sauvegarder vos fichiers et votre base de données car PHP-Fusion 7 est une mise à jour majeure par rapport aux précédentes. Vous devez suivre précisément les instructions suivantes.

1. Le code des versions n'est pas compatible directement avec la version 7. Vous devez soit mettre à jour les modifications ("mods"), infusions, paneaux ou thèmes avant la mise à jour OU vous procurer une mise à jour compatible avec la version 7. Il est recommandé de désactiver tout code d'autre provenance pour éviter les problèmes.

2. Commencez par uploader le contenu du répertoire 'upgrade v601x' dans le répertoire administration de votre site.

3. Connectez-vous en tant que Super Administrateur. Dans l'onglet Configuration du Site du Panneau d'Administration, cliquez sur Mise à Jour et exécutez la commande. VOUS DEVEZ suivre les indications jusqu'à ce que vous voyez la mention 'Database upgrade complete'.

4. TRES IMPORTANT : cette version comprenant des changements à la structure même du site, plusieurs éléments ne fonctionneront pas avant l'upload complet des fichiers de cette version. VOUS DEVEZ maintenant uploader TOUS les fichiers le contenu de php-files et tous ses sous-répertoires sur votre hébergeur.

Dans le doute, n'hésitez pas à contacter un de nos sites de support, vous trouverez dans les forums beaucoup d'utilisateurs avertis qui pourront vous conseiller.

CONSEILS DE SECURITE
Pour sécuriser votre site, nous vous conseillons de :
  • Vérifier que le fichier config.php n'a pas d'autorisation en écriture (CHMOD 644).
  • Ne jamais laisser setup.php sur le serveur après l'installation de PHP-Fusion.
  • Mettre des mots de passe différents au FTP et à la base MySQL.
  • Ne jamais autoriser des pièces jointes exécutables dans les forums, comme des fichiers php, html, exe, ou texte.
